# Учебная доска объявлений на Django

Простой учебный проект в стиле небольшой доски объявлений. Пользователи могут публиковать объявления, писать продавцам личные сообщения, вести несколько диалогов по одному объявлению и закрывать объявление после продажи с указанием итоговой цены.

## Функционал

- регистрация, вход и выход через стандартную систему аутентификации Django
- создание, просмотр, редактирование и удаление объявлений
- загрузка до 3 фотографий для каждого объявления
- отдельная страница профиля с никнеймом и аватаром
- личные сообщения между продавцом и покупателями
- несколько диалогов по одному объявлению
- закрытие объявления продавцом с указанием финальной цены продажи
- архив объявлений у владельца
- отзывы только после завершенной продажи
- поиск по названию и фильтрация по категории без перезагрузки страницы
- стандартная админка Django

## Стек технологий

- Django
- SQLite
- Django Templates
- обычный CSS
- чистый JavaScript (`fetch`)
- Pillow для загрузки изображений

## Бизнес-логика

### Объявления

- только автор может редактировать и удалять объявление
- к объявлению можно прикрепить до 3 фотографий
- объявление остается активным, даже если по нему уже написали один или несколько покупателей
- продавец может закрыть объявление вручную и указать финальную цену продажи
- после закрытия объявление попадает в архив владельца

### Переписка

- покупатель не может писать сам себе
- один покупатель может иметь только один диалог по одному объявлению
- по одному объявлению может быть несколько разных диалогов с разными покупателями
- если объявление уже закрыто, новые сообщения отправлять нельзя, но историю переписки можно читать

### Продажа

- при закрытии объявления продавец может выбрать, кому именно продал товар
- выбранный диалог помечается как завершенная продажа
- если товар продан не через сайт, продавец может закрыть объявление без выбора покупателя

### Отзывы

- отзыв можно оставить только после завершенной продажи
- отзыв может оставить только участник завершенной продажи
- один пользователь не может оставить второй отзыв по той же продаже

## Поиск и фильтрация

На главной странице есть поле поиска и выпадающий список категорий. При вводе текста или выборе категории отправляется AJAX-запрос через `fetch()`. Контейнер со списком объявлений обновляется без перезагрузки страницы. Во время загрузки показывается надпись `Загрузка...`.

## Как запустить

1. Установите Python 3.10+.
2. Создайте и активируйте виртуальное окружение.
3. Установите зависимости:

```bash
pip install -r requirements.txt
```

4. Примените миграции:

```bash
python manage.py migrate
```

5. Создайте администратора:

```bash
python manage.py createsuperuser
```

6. При желании добавьте категории через `/admin/`.
7. Запустите сервер:

```bash
python manage.py runserver
```

8. Откройте сайт в браузере: `http://127.0.0.1:8000/`

## Как запустить в Docker

1. Установите Docker Desktop.
2. Проверьте файл `.env` и при необходимости заполните переменные.
3. Соберите и запустите контейнер:

```bash
docker compose up --build
```

4. Откройте сайт в браузере: `http://127.0.0.1:8000/`

Что делает контейнер автоматически:

- применяет миграции
- собирает статику
- запускает Django через `gunicorn`
- хранит `db.sqlite3` и загруженные файлы в одном Docker volume
- может раздавать загруженные `media`-файлы для демо-режима

## Что проверить после запуска

- регистрация нового пользователя
- создание категории в админке
- создание объявления
- загрузка фотографий объявления
- изменение ника и аватара в профиле
- поиск и фильтрация без перезагрузки
- переписка нескольких покупателей с одним продавцом
- закрытие объявления с финальной ценой продажи
- появление закрытого объявления в архиве
- создание отзыва после завершенной продажи
